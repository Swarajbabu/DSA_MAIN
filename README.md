## 📦 Resources Included

### 📊 DSA Patterns Cheat Sheet (`DSA patterns Cheat Sheet.xlsx`)
A comprehensive reference sheet covering all major DSA problem-solving patterns — sliding window, two pointers, fast & slow pointers, merge intervals, cyclic sort, binary search variations, BFS/DFS, dynamic programming patterns, and more. Built from scratch during active problem-solving sessions.

### 📋 Interview Prep Tracker v2 (`DSA_Interview_Prep_v2.xlsx`)
A structured spreadsheet tracking problems by topic, difficulty, platform (LeetCode / GFG / HackerRank), status, and revision notes. Designed to make sure no topic gets missed before interviews.

### ❌ LeetCode Errors Log (`LEET CODE ERRORS.docx`)
A personal log of mistakes made while solving LeetCode problems — wrong approaches, edge cases missed, off-by-one errors, and the correct solutions. This is one of the most valuable files in the repo: learning from mistakes faster than re-solving problems.

### 💻 LeetCode Daily Solutions (`LeetCodeDailyQu.c++`)
Solutions to LeetCode daily challenge problems, written in C++ with comments explaining the approach and time/space complexity.

## 🎯 Problem-Solving Approach

Every problem in this repo follows a consistent approach before writing code:

```
1. Understand   →  Read carefully, identify input/output
2. Examples     →  Trace through 2-3 examples manually
3. Pattern      →  Identify which DSA pattern applies
4. Brute Force  →  Think of the naive O(n²) or O(n³) solution first
5. Optimise     →  Reduce to optimal time/space complexity
6. Code         →  Write clean, commented C++ solution
7. Test         →  Verify edge cases (empty, single element, negatives)
8. Log          →  Note any mistakes in LEET CODE ERRORS.docx
```
